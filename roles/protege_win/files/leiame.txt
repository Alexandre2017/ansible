Instalação:

-Extraia o conteudo do arquivo protege.zip para alguma pasta acessível pelo administrador.
-Dentro dos scripts, no segundo parametro HOME_ADM, defina a pasta onde os "snapshots" são salvos (por padrão salvos em c:\users\administrador)

########################################


Para proteger um usuário: (retorna as configurações atuais daquele após reiniciar máquina)

- Execute o script protege.cmd
- forneça o nome do usuário a proteger.
- O usuário fornecido é protegido.

Obs.: os seguintes arquivos e diretórios serão criados (se HOME_ADM=c:\users\administrador):

c:\users\administrador\backup\ln.exe		executável que efetua corretamente o backup dos arquivos.
c:\users\administrador\backup\nome-do-usuario\arq\	diretório com os arquivos do backup.
c:\users\administrador\backup\nome-do-usuario\reg\	diretório com backup do registro.
c:\users\administrador\backup\nome-do-usuario\lixeira\	diretório com backup da lixeira.
c:\users\administrador\backup\nome-do-usuario\job\	diretório com scripts da rotina de restauração.
c:\users\administrador\backup\nome-do-usuario\job\restaura-copia.cmd	script que restaura "snapshot"
c:\users\administrador\backup\nome-do-usuario\job\faz-copia.cmd 	script que cria "snapshot"
O script faz-copia.cmd pode ser usado para gerar uma versão mais atual a ser recuperada após os reinicios.

Um agendamento "restaura-padrao-nome-do-usuario" também é criado.

########################################


Para desproteger um usuário:

- Execute o script desprotege.cmd
- forneça o nome do usuário a desproteger
- O usuário fornecido é desprotegido e o seguinte diretório é apagado (se HOME_ADM=c:\users\administrador):
c:\users\administrador\backup\nome-do-usuario

O agendamento "restaura-padrao-nome-do-usuario" também é removido.
